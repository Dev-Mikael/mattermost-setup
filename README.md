# mattermost-kubeclaw

Automated GitOps deployment of **Mattermost** + **OpenClaw AI bot** on Kubernetes
using **FluxCD v2**, **Sealed Secrets**, **MetalLB**, **NGINX Ingress**,
**cert-manager**, the **Mattermost Kubernetes Operator**, and **KubeClaw**.

Supports both **kind** (local dev) and **kubeadm** (on-prem production).

---

## Quick Start

```bash
# 1. Fill in your values
cp .env.example .env
nano .env

# 2. On-prem only — run ON the server first:
# sudo bash scripts/02b-setup-kubeadm.sh

# 3. Run everything
bash bootstrap.sh
```

See **openclaw-bot-guide.md** for the full implementation guide and system explanation.

## Day-2 Operations

| Task | Command |
|------|---------|
| Upgrade Mattermost | Edit `version:` in `apps/mattermost/mattermost.yaml`, commit & push |
| Upgrade KubeClaw | Edit `version:` in `apps/kubeclaw/helmrelease.yaml`, commit & push |
| Manage bot (pairings, channels) | `bash scripts/07-manage-bot.sh` |
| Force Flux sync | `flux reconcile source git flux-system` |
| View all status | `flux get all -A` |
| Mattermost logs | `kubectl logs -n mattermost -l app=mattermost -f` |
| OpenClaw logs | `kubectl logs -n kubeclaw statefulset/kubeclaw -f` |
| All pods | `kubectl get pods -A` |

## DNS Setup (modumichael.me) — On-Prem Only

Create **two A records** at your DNS registrar:

| Host | Type | Value |
|------|------|-------|
| `@` | A | `YOUR_SERVER_PUBLIC_IP` |
| `openclaw` | A | `YOUR_SERVER_PUBLIC_IP` |

TLS certificates are issued automatically by Let's Encrypt via cert-manager.
