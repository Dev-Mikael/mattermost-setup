# mattermost-setup

Automated GitOps deployment of **Mattermost** on Kubernetes using **FluxCD v2**,
**Sealed Secrets**, **MetalLB**, **NGINX Ingress**, **cert-manager**, and the
**Mattermost Kubernetes Operator**.

Supports both **kind** (local dev) and **kubeadm** (on-prem production).

---

## Quick Start

```bash
# 1. Fill in your values
cp .env.example .env
nano .env

# 2. On-prem only — run ON the server first:
# sudo bash scripts/02b-setup-kubeadm.sh

# 3. Run everything
bash bootstrap.sh
```

See **Mattermost_Deployment_Guide.docx** for the full step-by-step guide.

## Day-2 Operations

| Task | Command |
|------|---------|
| Upgrade Mattermost | Edit `version:` in `apps/mattermost/mattermost.yaml`, commit & push |
| Force Flux sync | `flux reconcile source git flux-system` |
| View all status | `flux get all -A` |
| View Mattermost logs | `kubectl logs -n mattermost -l app=mattermost -f` |
| View all pods | `kubectl get pods -n mattermost` |

## DNS Setup (modumichael.me) — On-Prem Only

Create an **A record** at your DNS registrar:

| Host | Type | Value |
|------|------|-------|
| `@` | A | `YOUR_SERVER_PUBLIC_IP` |

TLS certificates are issued automatically by Let's Encrypt via cert-manager.
